AI security Report 01

Hidden AI Agents Create Access Risks

Security experts warned that organizations are accumulating hidden risks from orphaned AI agents and standing privileges left behind after employees change roles or leave. These autonomous tools often retain access to sensitive databases, source code repositories, and internal systems even after their creators’ accounts are revoked. Traditional security platforms struggle to identify these agents because they treat them like ordinary applications and lack ownership context. Researchers emphasized the importance of mapping AI identities to accountable human owners, discovering shadow AI deployments, and enforcing centralized governance. Failure to control dormant AI agents could create invisible attack paths and increase exposure to data breaches and insider threats.

threat roundup Report 02

Trusted Platforms Become Attack Vectors

The latest ThreatsDay bulletin highlights a surge in attacks exploiting trusted platforms and developer ecosystems. Researchers uncovered abuse of Anthropic Claude shared chats for malware delivery, malicious npm packages distributing the Rust-based NastyC2 framework, and Microsoft 365 device-code phishing campaigns bypassing password theft. Additional threats included fileless macOS malware, AI-themed phishing delivering AsyncRAT, malicious browser extensions, and Steam Workshop malware. Analysts observed attackers increasingly leveraging legitimate cloud services, coding agents, OAuth flows, and AI tools to evade detection. Security experts emphasized that trust itself has become an attack surface, requiring stronger validation, monitoring, and governance across digital ecosystems.

crypto malware Report 03

Clipboard Malware Steals Crypto Funds

Microsoft detailed a new Windows-based Clipper malware campaign designed to steal cryptocurrency by manipulating clipboard contents. Once installed, the malware continuously monitors the clipboard for copied cryptocurrency wallet addresses. When a victim copies an address to send funds, the malware silently replaces it with an attacker-controlled address, causing transactions to be redirected without obvious signs of compromise. Researchers observed the malware being distributed through malicious downloads, cracked software, and deceptive websites. The campaign highlights growing threats targeting cryptocurrency users and demonstrates how attackers exploit user trust and routine actions. Microsoft recommends verifying wallet addresses and maintaining updated endpoint protections.
