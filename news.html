malware delivery Report 01

Google Domain Abused For Malware Delivery

Researchers uncovered a malspam campaign abusing Google's DoubleClick advertising domain to deliver the DesckVB remote access trojan. The attack routes victims through a trusted Google-owned domain before redirecting them to attacker-controlled infrastructure, helping bypass security filters and reputation-based defenses. The campaign dynamically personalizes phishing pages using victim email addresses, company branding, and geographic details, increasing credibility and success rates. Once victims interact with the lure, malware is delivered, enabling remote access, surveillance, and potential credential theft. Analysts warn that abuse of legitimate cloud and advertising platforms continues to make phishing campaigns harder to detect and block using traditional security controls.

AI security Report 02

AI Unearths Hidden RCE Flaw

Security researchers reported that an autonomous AI-powered security tool successfully identified a remote code execution vulnerability that had remained undiscovered for nearly two years. The flaw existed in production software despite previous security reviews and testing efforts. The AI system autonomously analyzed code, traced execution paths, and identified the exploitable weakness without direct human guidance. Experts noted that the discovery highlights how advanced AI systems are increasingly capable of finding complex vulnerabilities that traditional methods overlook. While the technology offers significant defensive benefits, researchers warn that similar capabilities could also be adopted by attackers, accelerating vulnerability discovery and exploitation across software ecosystems.

windows vulnerability Report 03

Unpatched Windows Search URI bug exposes NTLMv2 hashes through malicious link interactions.

Security researchers disclosed an unpatched Windows vulnerability affecting the built-in Search URI handler that can expose NTLMv2 authentication hashes to attackers. The flaw abuses the search: URI mechanism to trigger outbound SMB authentication requests when a victim interacts with a specially crafted link. Attackers can capture leaked NTLMv2 hashes and potentially use them in relay attacks, enabling unauthorized access to systems and network resources. Researchers noted that the issue resembles a previously patched Windows URI handler vulnerability but currently lacks a CVE assignment and security update. Organizations are advised to restrict NTLM usage, monitor SMB traffic, and strengthen authentication controls while awaiting an official fix.
