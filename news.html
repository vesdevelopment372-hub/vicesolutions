North Korea Hits AI Supply Chain

Microsoft attributed the recent Mastra AI supply-chain attack to North Korea’s Sapphire Sleet, also known as BlueNoroff. Attackers hijacked an npm maintainer account and injected malicious updates into more than 140 packages using a typosquatted dependency named “easy-day-js.” When installed, the packages deployed malware targeting Windows, Linux, and macOS systems. The payload collected credentials, API keys, browser data, authentication tokens, and cryptocurrency wallet information while establishing persistence mechanisms. Researchers linked follow-on activity, PowerShell backdoors, and command infrastructure to previous Sapphire Sleet campaigns, highlighting North Korea’s continued focus on financial theft and software supply-chain compromises.

software vulnerability Report 02

WordPress SMTP Flaw Under Attack

Attackers are actively exploiting CVE-2026-4020 in the Gravity SMTP WordPress plugin to steal email-service credentials and sensitive system information. The flaw affects versions 2.1.4 and earlier and stems from an improperly protected REST API endpoint that allows unauthenticated users to retrieve detailed configuration reports. Exposed data may include API keys, OAuth tokens, server details, and information about installed plugins. Researchers observed widespread scanning activity involving hundreds of attacking IP addresses, while Wordfence blocked millions of exploitation attempts. Administrators are urged to update to Gravity SMTP 2.1.5 or later, rotate exposed credentials, and review logs for suspicious requests.

supply compromise Report 03

Klue Breach Hits Security Firms

A supply-chain attack targeting competitive intelligence platform Klue impacted multiple cybersecurity firms, including Huntress and other enterprise customers. Attackers compromised Salesforce environments linked to Klue services and exfiltrated customer-related information stored within integrated systems. Researchers indicated the breach involved unauthorized access rather than malware deployment, highlighting risks associated with interconnected SaaS ecosystems and third-party dependencies. The incident underscores how attackers increasingly target trusted service providers to reach downstream organizations. Affected companies initiated containment measures, reviewed access controls, and notified customers. Security experts emphasized stronger third-party risk management, SaaS monitoring, and identity protections to reduce exposure from supply-chain compromises affecting critical business platforms.
