cyber espionage Report 01

WinRAR Flaw Fuels Russian Espionage

Security researchers reported that multiple Russia-aligned threat groups continue exploiting a WinRAR vulnerability despite patches being available for nearly a year. The campaigns target Ukrainian organizations through phishing emails carrying malicious archive files that abuse CVE-2025-8088, a path traversal flaw. Successful exploitation enables deployment of malware families including data stealers, remote access tools, and self-propagating worms. Analysts observed attackers using the vulnerability to establish persistence, collect intelligence, and conduct long-term espionage operations. The continued success of the attacks highlights the risks posed by delayed patching and unmanaged software. Organizations are urged to update WinRAR immediately and strengthen phishing defenses.

backup vulnerability Report 02

Veeam Backup Servers Face Critical RCE

Veeam released emergency security updates for a critical remote code execution vulnerability affecting Backup & Replication version 12 deployments. Tracked as CVE-2026-44963 with a CVSS score of 9.4, the flaw allows authenticated domain users to execute arbitrary code on backup servers. Researchers warned that compromised backup infrastructure can provide attackers with privileged access to recovery systems, enabling ransomware deployment, data destruction, and broader network compromise. The vulnerability affects version 12.3.2.4465 and earlier 12.x builds but does not impact version 13 due to architectural changes. Organizations are strongly advised to upgrade immediately and review backup server access controls to reduce risk.

supply compromise Report 03

Microsoft Supply Chain Breach Hits AI Developers

Microsoft temporarily disabled dozens of GitHub repositories after attackers compromised open-source projects related to Azure and AI development tools. The malicious code targeted developers using platforms such as Claude Code, Gemini CLI, and VS Code, stealing passwords, authentication tokens, and other sensitive credentials when affected tools were opened. More than 70 repositories were reportedly impacted before Microsoft initiated containment measures and notified potentially affected users. Security researchers classified the incident as a software supply-chain attack, where trusted development resources are weaponized to compromise downstream users. The breach highlights growing threats against AI developers and open-source ecosystems increasingly targeted by credential theft campaigns.
